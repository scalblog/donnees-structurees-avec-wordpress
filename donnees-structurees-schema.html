Comment lesdonnées structurées de Schema.org impactent le référencement SEO.

Ce qui nous intéresse ici est d'aider les moteutrs de recherche à mieux comprendre le contenu d'une page web et, par conséquent, à mieux la remonter dans les résultats pertinents d'une requête.

John Mueller de chez Google a participé en 2015 à un tchat video où il a indiqué très clairement l'importance à venir des données structurées dans le domaine de l'optimisation SEO.
Le fait que Shema.org soit fondé et soutenu par des géants du Web conforte, si nous voulons contribuer à un Page Rank et une Indexation de nos pages corrects, qu'il faut inclure les données structurées à notre stratégie SEO.

Prenons un exemple. Un internaute recherche des "Places de concert pour Francis Cabrel". Google va lui retourner une liste de résultats et il y a fort à parier que les liens qui seront alors cliqués, même s'ils ne sont pas en tête de résultats, seront ceux qui offrent les informations les plus utiles.

Quel format de données structurées choisir ?

La documentation officielle de Google sur les données structurées fait la part belle au format JSON-LD. Nous tenons donc là une piste à privilégier. Google fournit même un Webmasters Tool qui génère les données au format JSON-LD, un outil qui peut être votre barrière d'entrée la plus accessible si vous débutez avec les données structurées.

Connaître les données qui peuvent être structurées

Le guide de Google sur les données structurées offre la ressource la plus exhaustive et la plus compréhensible.

La ligne de base par laquelle Schema veut catégoriser le contenu est le "type" de données et voici les types que Google reconnaît actuellement:

  • Articles
  • Books (livres)
  • Courses (cours, formation)
  • Datasets (ensembles de données)
  • Events (événements)
  • Fact Check
  • Job Postings (offres d'emploi)
  • Local Businesses (entreprises locales)
  • Music (musique)
  • Podcasts
  • Products (produits)
  • Recipes (recettes de cuisine)
  • Reviews (critiques, analyses)
  • TV & Movies (Films et télévision)
  • Videos

En complément, Google cherche aussi des données structurées dans ce qui améliore l'expérience utilisateur:

  • Fil d'Ariane (Breadcrumb)
  • Barre de recherche dans le site
  • Informations de contact de la société
  • Logos
  • Liens vers les pages de Réseaux sociaux
  • Carousels

Ajouter les données structurées de Schema.org à votre Wordpress

Pour cette démonstration nous allons utiliser comme extension Wordpress le plugin Advanced Custom Fields (ACF). Il nous permettra au final de générer les données structurées dans un format JSON-LD valide.

Certains lecterus seront tentés de demander pourquoi ne pas utiliser plutôt un plugin populaire comme Yoast ou Schema. En fait, il existe des tonnes d'extensions qui aident à structurer les données sous Wordpress.
Je vous laisse vous faire votre propre opinion mais à titre personnel je recherche une extension qui permet de manipuler les données structurées avec un niveau de personnalisation assez poussé pour pouvoir manipuler notamment des données comme les heures d'ouverture et les informations de contact d'une entreprise locale.

Avec ACF nous avons non seulement la possibilité de créer les champs exactement voulus pour la saisie des valeurs que nous déisrons mais nous pouvons en plus le faire dynamiquement comme une donnée courante de notre site Wordpress.

#Etape 4 : Saisir les données

Nous avons bien préparé le terrain, saisons à présent les fameuses données structurées que Google apprécie tant.

  • Logo de l'entreprise : Google spécifie que la taille idéale est un carré de 151px de côté. Google utilisera cette image s'il affiche les informations de l'entreprise sur la droite de la page de ses résultats. Vous pouvez en voir un exemple en cherchant une entreprise bien connue, comme Google par exemple.
  • Photo de l'entreprise locale : Là aussi cela peut améliorer l'affichage dans la partie droite des résultats d'une recherche dans Google. Cela peut aussi améliorer les résultats dans une recherche sur Maps. Google recommande une image carrée de 200px.
  • Type de donnée structurée : Sélectionnez le type de contenu dans ceux identifiés par Google. Ici il s'agira de "local business".
  • Adresse de l'entreprise : Là aussi ce sera utilisé sur la partie qui présente l'entreprise dans les résultats de recherche que dans Maps.
  • Horaires d'ouverture : Les spécifications popur les heures d'ouverture peuvent être consultées sur le site de Schema.org. La façon dont nous renseignons ici le champs utilise la fonction Repeater de la version payante d'ACF pour spécifier les jours d'ouverture, les heures d'ouverture et de fermeture et pour basculer entre ouverture et fermeture selon l'heure. Cela devrait courvir nos besoins, si l'on en croit la documentation de Schema.org.
  • Jours de fermeture spéciale : cela indiquera les jours spéciaux de fermeture. C'est l'avantage "cool" d'ACF pour rendre service aux internautes qui feront leur recheche lors d'un jour férié.
